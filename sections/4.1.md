# 4.1 Prototypowanie UI z Angular CLI i Angular Material: Od opisu tekstowego do gotowego komponentu Angular

**Moduł 4: Szybkie Wdrażanie (Ship Fast) w oparciu o Angular**

---

## Teoria

### 1. Filozofia „Ship Fast" — dlaczego szybkie prototypowanie ma znaczenie?

W świecie AI-first development tempo jest kluczowe. Zanim piszesz jakikolwiek kod, warto zrozumieć, że **prototyp to nie produkt** — jego celem jest jak najszybsze uwidocznienie idei, zebranie feedbacku i weryfikacja założeń. Dopiero po potwierdzeniu wartości warto inwestować w dopracowanie architektury.

W tym kontekście **Angular CLI** i **Angular Material** tworzą zgrany tandem:
- Angular CLI eliminuje konfigurację od zera — jedno polecenie generuje gotową strukturę projektu, komponenty, serwisy i routing.
- Angular Material dostarcza gotowe, dostępne (a11y) komponenty UI zgodne z Material Design, bez konieczności pisania CSS od podstaw.

Razem pozwalają przejść od pustego folderu do działającego interfejsu w ciągu minut — nie godzin. To właśnie jest esencja „Ship Fast".

---

### 2. Angular CLI — pierwsze kroki i najważniejsze komendy

**Angular CLI** (`@angular/cli`) to narzędzie wiersza poleceń zarządzające całym cyklem życia projektu Angular: tworzeniem, generowaniem kodu, uruchamianiem, testowaniem i budowaniem.

**Instalacja i tworzenie projektu:**

```bash
# Instalacja CLI (jednorazowo)
npm install -g @angular/cli

# Nowy projekt ze standalone components i SCSS
ng new my-ai-app --standalone --style=scss --routing=true

cd my-ai-app
ng serve   # uruchamia dev server na http://localhost:4200
```

**Najważniejsze komendy generowania:**

| Komenda | Skrót | Efekt |
|---------|-------|-------|
| `ng generate component nazwa` | `ng g c nazwa` | Nowy komponent (ts, html, scss, spec) |
| `ng generate service nazwa` | `ng g s nazwa` | Nowy serwis |
| `ng generate pipe nazwa` | `ng g p nazwa` | Nowy pipe |
| `ng generate interface nazwa` | `ng g i nazwa` | Nowy interfejs TypeScript |
| `ng build --configuration production` | — | Build produkcyjny z optymalizacją |

Flaga `--dry-run` (`-d`) pozwala podejrzeć, jakie pliki zostaną wygenerowane, bez ich faktycznego tworzenia — niezastąpiona przy nauce.

```bash
ng g c features/chat-window --dry-run
```

### 3. Angular Material — biblioteka komponentów i theming

**Angular Material** to oficjalna biblioteka komponentów UI dla Angular, oparta na specyfikacji Material Design od Google. Zawiera ponad 35 gotowych komponentów: przyciski, formularze, karty, tabele, dialogi, nawigację i wiele więcej.

**Instalacja schematem (automatycznie konfiguruje moduły i style):**

```bash
ng add @angular/material
# Kreator zapyta o: motyw kolorystyczny, typografię i animacje
```

Polecenie `ng add` to więcej niż zwykły `npm install` — wykonuje **schemat** (ang. *schematic*), który automatycznie importuje potrzebne moduły, dodaje style do `angular.json` i konfiguruje animacje.

**Przykład użycia komponentów Material w standalone component:**

```typescript
// chat-window.component.ts
import { Component } from '@angular/core';
import { MatCardModule } from '@angular/material/card';
import { MatFormFieldModule } from '@angular/material/form-field';
import { MatInputModule } from '@angular/material/input';
import { MatButtonModule } from '@angular/material/button';
import { FormsModule } from '@angular/forms';

@Component({
  selector: 'app-chat-window',
  standalone: true,
  imports: [MatCardModule, MatFormFieldModule, MatInputModule, MatButtonModule, FormsModule],
  template: `
    <mat-card>
      <mat-card-header>
        <mat-card-title>Asystent AI</mat-card-title>
      </mat-card-header>
      <mat-card-content>
        <mat-form-field appearance="outline" class="full-width">
          <mat-label>Twoje pytanie</mat-label>
          <textarea matInput [(ngModel)]="query" rows="3"></textarea>
        </mat-form-field>
      </mat-card-content>
      <mat-card-actions>
        <button mat-raised-button color="primary" (click)="send()">Wyślij</button>
      </mat-card-actions>
    </mat-card>
  `
})
export class ChatWindowComponent {
  query = '';
  send() { console.log(this.query); }
}
```

### 4. Od opisu tekstowego do komponentu — workflow z pomocą AI

Kluczową umiejętnością w AI-first development jest **skuteczne opisanie komponentu** tak, by narzędzie AI (GitHub Copilot, Cursor, Claude) wygenerowało gotowy kod Angular Material od razu nadający się do użycia.

**Dobry prompt dla AI zawiera:**
- typ komponentu (formularz, lista, karta, dialog),
- bibliotekę (`Angular Material`),
- zachowanie (reaktywny formularz, walidacja, binding),
- pola i ich typy,
- opcjonalnie: styl układu (`flex`, `grid`, `mat-grid-list`).

**Przykład promptu:**
> „Wygeneruj standalone Angular component z Angular Material: formularz rejestracyjny z polami Email (mat-input, walidacja email), Hasło (mat-input type=password, min 8 znaków), przyciskiem Submit (mat-raised-button color=primary). Użyj ReactiveFormsModule. Jeśli formularz jest niepoprawny, przycisk ma być wyłączony."

Z takiego opisu AI generuje kompletny, działający komponent. Twoja rola to:
1. ocena czy komponent spełnia wymagania,
2. drobna korekta (nazwy, style),
3. integracja z resztą aplikacji.

Zasada **review, don't retype** — nie przepisujesz, weryfikujesz i adaptujesz.

### 5. Schematy (Schematics) i generowanie boilerplate'u przy użyciu AI

Angular CLI opiera się na mechanizmie **schematów** — skryptów transformacji kodu, które mogą tworzyć i modyfikować pliki projektu. Możesz tworzyć własne schematy lub korzystać z gotowych dostarczanych przez biblioteki (np. `ng add @angular/material`).

W kontekście prototypowania AI może wygenerować polecenia CLI do stworzenia pełnej struktury funkcjonalności:

```bash
# Wygenerowane przez AI polecenia dla modułu "chat"
ng g c features/chat/chat-shell --standalone
ng g c features/chat/message-list --standalone
ng g c features/chat/message-input --standalone
ng g s features/chat/chat
ng g i features/chat/message
```

Zamiast ręcznie tworzyć każdy plik, opisujesz funkcjonalność w języku naturalnym, a AI tłumaczy to na sekwencję komend CLI. To dramatycznie przyspiesza inicjalizację nowych modułów.

### 6. Lokalne środowisko deweloperskie z Dockerem

Podczas prototypowania warto uruchomić całe środowisko (frontend + backend + baza) za pomocą Dockera — dzięki temu nowy deweloper może wystartować jedną komendą bez konfigurowania środowiska ręcznie.

```dockerfile
# Dockerfile.dev — tylko dla developmentu!
FROM node:22-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
EXPOSE 4200
CMD ["npx", "ng", "serve", "--host", "0.0.0.0", "--poll", "500"]
```

```yaml
# docker-compose.yml (development)
services:
  frontend:
    build:
      context: .
      dockerfile: Dockerfile.dev
    ports:
      - "4200:4200"
    volumes:
      - .:/app
      - /app/node_modules   # anonimowy volume chroni node_modules hosta

  backend:
    build: ./backend
    ports:
      - "3000:3000"
    environment:
      - MONGODB_URI=mongodb://mongo:27017/aiapp

  mongo:
    image: mongo:7
    ports:
      - "27017:27017"
```

```bash
docker compose up --build   # start całego stacku deweloperskiego
```

**Ważne**: ten Dockerfile jest przeznaczony wyłącznie do developmentu — w produkcji używamy AWS Lambda do backendu, a frontend trafia jako statyczny bundle na S3/CloudFront.

### 7. Najlepsze praktyki przy szybkim prototypowaniu

- **Standalone components** (Angular 17+) — brak NgModules, prostszy import, szybszy start.
- **Lazy loading** od pierwszego dnia — nawet w prototypie zadbaj o podział kodu (`loadComponent`), by uniknąć refaktoryzacji przed produkcją.
- **Angular CDK Layout** (`BreakpointObserver`) — responsywność bez media queries w CSS.
- **`ng serve --open`** — automatyczne otwieranie przeglądarki przy starcie dev serwera.
- **Skróty CLI z aliasami Bash** — przyspiesza powtarzalne operacje:

```bash
# ~/.bashrc lub ~/.zshrc
alias ngs='ng serve --open'
alias ngc='ng generate component'
alias ngb='ng build --configuration production'
```

- **Hot Module Replacement** jest domyślnie włączony w Angular 17+ (`ng serve`) — zmiany w komponentach widać natychmiast bez pełnego przeładowania.

## Zadania praktyczne

### Zadanie 1 — Łatwe
**Projekt startowy z Angular Material**

Utwórz nowy projekt Angular za pomocą `ng new` (standalone, SCSS, z routingiem). Dodaj Angular Material poleceniem `ng add`, wybierając dowolny predefiniowany motyw. Wygeneruj komponent `HomeComponent` i umieść w nim kartę (`mat-card`) z tytułem, krótkim opisem i przyciskiem nawigacyjnym. Uruchom aplikację i zweryfikuj jej działanie w przeglądarce.

---

### Zadanie 2 — Średnie
**Formularz chat z walidacją reaktywną**

Korzystając z GitHub Copilot lub innego narzędzia AI, wygeneruj standalone komponent zawierający reaktywny formularz (`ReactiveFormsModule`) z:
- polem tekstowym `mat-textarea` na pytanie (wymagane, min. 10 znaków),
- selectem `mat-select` z listą dostępnych modeli AI (np. GPT-4o, Claude 3.5, Llama 3),
- przyciskiem Submit (`mat-raised-button`) wyłączonym gdy formularz jest niepoprawny.

Po kliknięciu Submit wyświetl wartości formularza w konsoli. Dodaj wizualny komunikat błędu (`mat-error`) pod każdym polem.

---

### Zadanie 3 — Średnie
**Środowisko deweloperskie z Docker Compose**

Skonfiguruj środowisko `docker-compose.yml` dla aplikacji deweloperskiej składającej się z trzech serwisów: Angular frontend (Dockerfile.dev z hot-reload), Node.js backend (prosty Express serwujący `GET /api/health`), MongoDB. Upewnij się, że:
- zmiany w plikach Angular są widoczne bez restartu kontenerów (volume mount),
- backend łączy się z MongoDB przez wewnętrzną sieć Docker,
- komenda `docker compose up` startuje cały stack bez dodatkowej konfiguracji.

---

### Zadanie 4 — Zaawansowane
**Generator komponentów AI-assisted z pełnym przepływem**

Opisz w języku naturalnym (po polsku lub angielsku) trzy różne komponenty UI dla aplikacji AI-chatbota: listę wiadomości z podziałem na role użytkownik/asystent, panel ustawień z suwakami parametrów (temperature, max_tokens) oraz widok historii sesji z wyszukiwaniem. Użyj AI (GitHub Copilot, Cursor lub Claude) do wygenerowania kodu każdego komponentu. Następnie:
1. Zintegruj wszystkie trzy komponenty w spójny routing Angular (`/chat`, `/settings`, `/history`).
2. Zastosuj Angular Material Navigation (`mat-sidenav`, `mat-toolbar`) jako shell aplikacji.
3. Dodaj motyw niestandardowy Angular Material (własna paleta kolorów primary/accent) zdefiniowany w SCSS.
4. Skonfiguruj lazy loading dla każdej trasy.
5. Uruchom całość w środowisku Docker Compose z backendem Node.js zwracającym mockowane dane.
